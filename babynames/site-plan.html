Baby Name Finder

gentle ideas • favorites • compare

Purpose

Baby Name Finder helps expecting parents discover, filter, and save baby name ideas. The site provides meanings, origins, popularity, and simple tools to compare and favorite names.

Audience

Expecting parents, family members, and anyone helping choose baby names. Audience values simple UX, cute pastel visuals, quick search and privacy (favorites stored locally).

Dynamic Elements

  • Random name generator (uses a function that picks a random index).
  • Filter system (gender, length, origin, popularity) using Array.filter.
  • Autosuggest search that shows possible names as you type.
  • Favorites saved in localStorage (add/remove favorites).
  • Compare view to place two names side-by-side.
  • “Battle” mode (pick one of two random names) to build favorites by selection.
  • Module structure: ES module for name dataset, main script for UI logic.

Required JS/Technical Items

  • Objects for each name (name, gender, meaning, origin, popularity, tags).
  • ES Module with exported `names` array.
  • Array methods used: forEach, map, filter, find, sort.
  • DOM interactions: querySelector, event listeners, classList, innerHTML.
  • Conditional logic to handle filtering, favorites, and UI states.

Logo • Colors • Fonts

Logo

Soft square icon with pastel gradient and simple marks representing name tags.

Colors

  • Primary: Mint — #C6F0E6
  • Accent: Baby Pink — #FFD7D7
  • Soft: Cream — #FFF7EA
  • Text: Slate — #2F3A3A

Fonts

Primary: Inter or system sans-serif for body. Headings use a friendly rounded font such as "Nunito" or system fallback.

Pages & Content

Home (index.html)

  • Hero with random name generator and "Surprise me" button
  • Search bar with autosuggest
  • Filter controls: gender, length, first letter, style (classic/modern)
  • Grid of name cards (image placeholder, name, meaning, tags, favorite button)

Child page: Favorites (favorites.html)

  • List of favorited names stored in localStorage
  • Option to remove or export favorites (JSON copy)

Child page: Compare (compare.html)

  • Choose two names to compare meaning, origin, popularity
  • Accessible layout, keyboard navigable

Accessibility & Performance Plan

  • Use semantic HTML and proper aria labels for interactive widgets.
  • Keyboard accessible controls (filters, favorites, compare).
  • Images use alt text; decorative SVGs marked appropriately.
  • Lazy-load large images and keep asset sizes small for Lighthouse performance.
